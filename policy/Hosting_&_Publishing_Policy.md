# ПРАВИЛА ЗАГРУЗКИ И ПУБЛИКАЦИИ ПРОЕКТОВ (Hosting & Publishing Policy)

**Дата вступления в силу:** 01.08.2026

Настоящие правила регулируют загрузку, хранение и публикацию сайтов и программных проектов через Сервис.

---

# 1. Общие требования

Каждый опубликованный проект должен:

* соответствовать законодательству;
* не нарушать права третьих лиц;
* не создавать угрозу безопасности;
* не нарушать правила используемой инфраструктуры.

---

# 2. Разрешённые проекты

Разрешены:

* персональные сайты;
* портфолио;
* блоги;
* документация;
* образовательные проекты;
* веб-приложения;
* открытый исходный код;
* тестовые проекты.

---

# 3. Запрещённые проекты

Запрещены сайты:

## Безопасность

* панели управления чужими системами;
* эксплойты;
* инструменты атак;
* вредоносные загрузчики.

---

## Мошенничество

Запрещены:

* фальшивые страницы банков;
* кража аккаунтов;
* поддельные магазины;
* мошеннические формы оплаты.

---

## Незаконные материалы

Запрещены:

* нелегальный контент;
* нарушение авторских прав;
* распространение украденного ПО;
* утечки данных.

---

# 4. Ограничения ресурсов

Сервис может ограничивать:

* количество запросов;
* использование CPU;
* объём хранения;


Использование ресурсов для нарушения работы сервиса запрещено.

---

# 5. Пользовательский код

Пользователь обязан убедиться, что:

* имеет право использовать код;
* соблюдает лицензии;
* не загружает чужие секреты.

Запрещается публиковать:

* API-ключи;
* пароли;
* приватные токены;
* закрытые ключи.

---

# 6. Жалобы и удаление

При получении обоснованной жалобы сервис может:

* временно скрыть проект;
* запросить объяснения;
* удалить нарушающий материал.

Жалобы могут касаться:

* авторских прав;
* нарушения закона;
* безопасности.

---

# 7. Ответственность владельца проекта

Владелец проекта отвечает за:

* содержание сайта;
* действия пользователей своего сайта;
* используемый код;
* соблюдение лицензий.

---

# 8. Прекращение размещения

Сервис может удалить проект без предварительного уведомления, если:

* существует угроза безопасности;
* проект нарушает закон;
* проект нарушает правила Cloudflare/GitHub;
* проект используется для атак.
